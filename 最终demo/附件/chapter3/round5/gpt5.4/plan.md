## 执行Plan

### Step 1: 明确报告目标与高管阅读口径
- 目标：
  - 将报告定位为“给CEO/CFO看的决策型行业分析”，不是技术科普，也不是泛泛趋势综述。
  - 明确核心问题：制造业AI当前到了什么阶段、哪些场景最值得投、ROI如何判断、未来2—3年怎么布局。
  - 统一篇幅与结构边界，确保全文控制在8000字左右。
- 需要调用的工具/数据源：
  - 任务说明
  - 过往行业报告常见结构
  - 高管读者偏好框架：市场规模、渗透率、降本增效、资本开支、回报周期、风险敞口
- 预计产出：
  - 1页写作Brief
  - 报告核心问题清单
  - 目标读者画像与表达原则
  - 拟定总标题、副标题、摘要方向
- 前置依赖：
  - 无

### Step 2: 搭建报告总框架与论证主线
- 目标：
  - 搭建适合高管快速阅读的目录结构，形成“现状—场景—价值—挑战—趋势—建议”的主线。
  - 将8000字拆分到各章节，避免后期内容失衡。
- 需要调用的工具/数据源：
  - 行业研究报告写作模板
  - 咨询公司报告结构样例
  - 制造业数字化/智能制造政策文本目录
- 预计产出：
  - 一级、二级目录
  - 每章核心论点、关键数据位、案例位
  - 各章节字数预算
- 前置依赖：
  - Step 1完成的写作Brief

### Step 3: 制定数据口径与证据标准
- 目标：
  - 提前定义哪些数据可以进入正文，避免后续资料杂乱、口径不统一。
  - 建立“权威优先、近两年优先、可交叉验证优先”的证据标准。
- 需要调用的工具/数据源：
  - 国家统计局
  - 工信部
  - 发改委
  - 中国信通院
  - 中国电子学会
  - 世界经济论坛
  - 麦肯锡、德勤、IDC、Gartner、BCG等公开报告
  - 上市公司年报/投资者交流纪要
- 预计产出：
  - 数据源优先级清单
  - 数据引用规则
  - 关键指标表：市场规模、渗透率、投资强度、效率提升、良率改善、库存周转、人工替代、回收周期
- 前置依赖：
  - Step 2完成的目录框架

### Step 4: 完成宏观与行业层数据搜集
- 目标：
  - 收集中国制造业总体运行数据，以及AI、工业软件、自动化、工业互联网、机器人、智能工厂等相关数据。
  - 建立“制造业AI应用现状”的宏观事实基础。
- 需要调用的工具/数据源：
  - 国家统计局制造业增加值、利润、投资、就业数据
  - 工信部智能制造试点示范、两化融合、工业互联网相关数据
  - 信通院人工智能、工业互联网白皮书
  - 公开券商研报与咨询机构数据
- 预计产出：
  - 宏观数据表
  - 5—8条可直接写入正文的关键结论
  - 适合高管阅读的“行业现状判断”
- 前置依赖：
  - Step 3的数据口径与证据标准

### Step 5: 梳理制造业AI应用场景与价值链地图
- 目标：
  - 将AI应用按制造业价值链拆解，突出哪些场景已经落地、哪些仍在试点、哪些具备未来增量。
  - 重点覆盖研发、采购、生产、质检、设备运维、仓储物流、供应链计划、销售预测、能源管理、管理决策等环节。
- 需要调用的工具/数据源：
  - 智能制造案例集
  - 龙头企业公开案例
  - 行业白皮书
  - 学术与产业会议资料
- 预计产出：
  - “场景—价值—成熟度”矩阵
  - 高优先级场景清单
  - 可写成正文小节的场景归纳材料
- 前置依赖：
  - Step 4完成的行业现状数据

### Step 6: 提炼头部企业案例与财务价值证据
- 目标：
  - 选择6—8个代表性案例，覆盖离散制造与流程制造，体现CEO/CFO最关心的财务与经营指标。
  - 尽量用公开、可验证的数据说明AI在降本、提效、增收、控风险方面的作用。
- 需要调用的工具/数据源：
  - 上市公司年报
  - 企业官网案例
  - 投资者关系纪要
  - 媒体专访
  - 行业奖项/示范工厂材料
- 预计产出：
  - 案例库
  - 案例摘要卡片
  - 财务价值证据表，如良率提升、停机时间下降、库存优化、人工节省、交付周期缩短
- 前置依赖：
  - Step 5完成的场景地图

### Step 7: 形成趋势判断与高管决策观点
- 目标：
  - 在现状基础上输出未来2—3年的趋势判断，而不是泛泛描述“AI前景广阔”。
  - 从技术、产业、资本、组织、监管五个维度提炼趋势。
- 需要调用的工具/数据源：
  - 最新政策
  - 行业投资与融资动向
  - 大模型在工业领域的应用进展
  - 海外制造业AI实践对比资料
- 预计产出：
  - 5—7条趋势判断
  - 每条趋势对应的驱动因素、约束因素、对企业经营的影响
  - “CEO/CFO应关注什么”的观点清单
- 前置依赖：
  - Step 4至Step 6的事实与案例基础

## 关键决策点
1. 报告主线是偏“行业扫描”还是偏“投资决策建议”。建议以后者为主，因为CEO/CFO更关注资源配置与回报判断。
2. 数据口径采用“近两年公开权威数据+经典案例长期趋势”的组合，而不是追求所有数字都最新但来源不稳。
3. 场景选择是“覆盖全链条”还是“聚焦高价值环节”。建议聚焦生产、质检、设备运维、供应链计划四类高ROI场景。
4. 案例选择是“知名大厂”还是“行业可复制样本”。建议兼顾龙头标杆与可复制案例，避免报告只停留在头部企业叙事。
5. 趋势部分是写技术演进还是经营影响。建议以经营影响为主，技术演进只保留与决策强相关内容。
6. 建议部分是原则性建议还是分阶段动作。建议写成“短期试点—中期扩面—长期体系化”的三阶段建议。

## 风险与备选方案
- 风险1：公开数据分散、口径不一致，导致结论不稳 → 备选：优先采用国家部委、信通院、上市公司年报等高可信来源，并对关键数据做双重交叉验证。
- 风险2：制造业AI案例很多，但可量化ROI的公开材料不足 → 备选：用“经营指标改善”替代绝对财务收益，必要时采用区间表达和行业平均估算，并明确标注数据来源和计算假设。
- 风险3：内容容易写成技术报告，不够适合CEO/CFO → 备选：每章都设置“经营含义”与“管理启示”段落，减少算法细节，增加投入产出、组织能力与风险控制视角。
- 风险4：8000字篇幅内信息过载，影响可读性 → 备选：采用“结论前置+图表承载细节+案例短卡片”写法，把冗长背景压缩到最少。
- 风险5：3天周期紧，后期可能来不及补足所有案例 → 备选：优先完成主线论证与核心数据，案例先保证4个高质量样本，再视时间补充到6—8个。
- 风险6：趋势判断容易流于空泛 → 备选：每个趋势必须对应至少1个数据事实、1个案例证据或1个政策/资本信号。

## 你认为最难的一步是什么？为什么？
最难的是Step 6“提炼头部企业案例与财务价值证据”。

原因在于，制造业AI案例很多，但真正对CEO/CFO有说服力的不是“企业用了AI”，而是“AI带来了多大经营改善、回收周期如何、能否复制”。公开材料往往会强调示范意义，却不一定完整披露投入规模、收益口径和基准对照。因此，这一步既要保证案例典型性，又要尽量找到可验证的量化证据，还要避免夸大效果。它直接决定整份报告是否具有决策参考价值，而不是停留在趋势描述层面。
